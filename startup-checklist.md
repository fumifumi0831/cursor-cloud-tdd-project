# Cursor Cloud Agent TDD環境：スタートアップチェックリスト

## 📋 事前準備（所要時間：10分）

### アカウント準備
- [ ] GitHubアカウントがある
- [ ] クレジットカードがある（Cursor Pro用）
- [ ] メールアドレスがある（各種サービス登録用）

### ローカル環境確認
- [ ] Node.js v20以上がインストール済み（`node --version`）
- [ ] npm v10以上がインストール済み（`npm --version`）
- [ ] Gitがインストール済み（`git --version`）
- [ ] GitHub CLI（gh）がインストール済み（オプション）

## 🚀 セットアップ（所要時間：30分）

### フェーズ1: Cursor準備（5分）
- [ ] Cursorをダウンロード＆インストール
- [ ] Cursor Proに加入（$20/月）
- [ ] Privacy Modeをオフに設定
- [ ] Background Agentsを有効化

### フェーズ2: GitHub準備（5分）
- [ ] 新規リポジトリ作成
- [ ] CursorとGitHub連携
- [ ] リポジトリへの読み書き権限付与

### フェーズ3: データベース準備（5分）
- [ ] Supabaseアカウント作成
- [ ] 新規プロジェクト作成（dev環境）
- [ ] データベースパスワード設定（保存！）
- [ ] 接続文字列取得

### フェーズ4: Cursor設定（10分）
- [ ] Cursor Secretsに DATABASE_URL 登録
- [ ] .cursor/environment.json 作成
- [ ] プロジェクトに jest をインストール
- [ ] サンプルテストファイル作成

### フェーズ5: 動作確認（5分）
- [ ] ローカルでテスト実行成功
- [ ] データベース接続テスト成功
- [ ] Gitコミット＆プッシュ
- [ ] Cloud Agent初回実行テスト

## ✅ 本番運用前の最終チェック

### セキュリティ
- [ ] .env ファイルが .gitignore に含まれている
- [ ] パスワードがコードに直接書かれていない
- [ ] Cursor Secrets が正しく設定されている
- [ ] GitHubリポジトリの公開範囲が適切

### テスト環境
- [ ] テストが独立して実行できる
- [ ] テストデータのクリーンアップが適切
- [ ] 並列実行時の競合がない
- [ ] タイムアウト設定が適切

### Cloud Agent設定
- [ ] environment.json がリポジトリにコミット済み
- [ ] install スクリプトが冪等性を保っている
- [ ] terminals 設定が適切
- [ ] 環境変数が正しく設定されている

## 🎯 初回実行のための簡易テスト

以下のコマンドがすべて成功すればOK：

```bash
# 1. ローカルテスト実行
npm test
# 期待される結果: All tests passed

# 2. データベース接続確認
DATABASE_URL="your-connection-string" npm test
# 期待される結果: Database tests passed

# 3. Git状態確認
git status
# 期待される結果: nothing to commit, working tree clean

# 4. リモート確認
git remote -v
# 期待される結果: GitHub URLが表示される
```
